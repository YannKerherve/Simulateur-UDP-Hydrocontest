Télécharger et ouvrir le logiciel

Téléchargez le fichier Simulateur_Hydrocontest.exe et double-cliquez dessus pour le lancer.

Capture de l'icône du simulateur telle qu'elle apparaît dans l'explorateur Windows
⚠️
Si Windows affiche un avertissement SmartScreen, cliquez sur « Informations complémentaires » puis « Exécuter quand même ».
2
Les deux fenêtres qui s'ouvrent

Au démarrage, deux fenêtres apparaissent automatiquement :

  1. La fenêtre logicielle — c'est la télécommande. Elle affiche les 6 drones avec leurs leviers de vitesse.
  2. Une page web dans votre navigateur — elle affiche la carte OpenStreetMap de la Rade Sud de Marseille avec les bateaux en temps réel.
La fenêtre principale avec les 6 cartes drones en ligne et la barre de contrôle en haut
La carte Leaflet/OpenStreetMap dans le navigateur avec les 6 triangles de bateaux positionnés sur la rade
ℹ️
Si la carte ne s'ouvre pas automatiquement, cliquez sur le bouton 🗺 OUVRIR CARTE en haut à droite de la fenêtre logicielle.
3
Démarrer la simulation

Cliquez sur le bouton ▶ DÉMARRER TOUS — le grand bouton vert en haut à gauche de la fenêtre logicielle.

Zoom sur la barre de titre avec le bouton « DÉMARRER TOUS » en vert et les boutons ARRÊTER / RESET

Les 6 bateaux sont alors prêts à naviguer. Ils démarreront dès que vous agirez sur leurs leviers de vitesse individuels.

4
Connecter chaque drone en UDP OUT

Pour chaque drone, dans sa carte :

  1. Vérifiez que le protocole est bien réglé sur UDP OUT (c'est la valeur par défaut).
  2. Si le simulateur et le serveur hydro_server.py tournent sur la même machine, l'IP affichée 127.0.0.1 convient. Sinon, saisissez l'IP réseau du serveur.
  3. Cliquez sur CONNECT. Le bouton passe en vert.
Une carte drone avec UDP OUT sélectionné, IP 127.0.0.1, bouton DISCONNECT vert — et le levier poussé à mi-course

5
Lancer hydro_server.py

Ouvrez le dossier hydro_contest et lancez hydro_server.py avec Thonny :

  1. Ouvrir Thonny Python
  2. Ouvrir le fichier hydro_server.py via Fichier → Ouvrir
  3. Cliquer sur Exécuter le script courant (flèche blanche sur rond vert, ou F5)
Thonny avec hydro_server.py ouvert, curseur sur le bouton Run, et les logs qui apparaissent dans la console en bas

Les logs suivants doivent apparaître dans la console :

INFO - Creating drones
INFO - MAVLink Comms: 1 : 14551
INFO - MAVLink Comms: 2 : 14552
INFO - MAVLink Comms: 3 : 14553
INFO - MAVLink Comms: 4 : 14554
INFO - Starting NMEA2000 TCP server for Drone 1 on port 5001
...
INFO - Web GUI is running on http://0.0.0.0:8080

Rendez-vous ensuite dans votre navigateur à l'adresse :

http://localhost:8080/
✅
Les données simulées (position, vitesse, courant, énergie) apparaissent sur l'interface web. Les bateaux se déplacent en fonction des leviers actionnés dans le simulateur.
6
Tester avec OpenCPN (NMEA / AIS)

Pour tester le flux NMEA vers les postes TimeZero des équipes, utilisez le logiciel gratuit OpenCPN.

  1. Ouvrir OpenCPN
  2. Aller dans Options → Connexions → Ajouter une connexion
  3. Choisir le protocole TCP
  4. Saisir l'adresse IP : 127.0.0.1
  5. Saisir le port : 5001 (pour le Drone 1, 5002 pour le Drone 2, etc.)
  6. Cocher Recevoir en entrée sur ce port
  7. Valider avec OK
Fenêtre de configuration de connexion dans OpenCPN : TCP, 127.0.01, port 5001, option « Recevoir en entrée » cochée

Le bateau simulé apparaît sur la carte OpenCPN, ainsi que les cibles AIS des autres drones.

Carte OpenCPN avec le bateau principal et les cibles AIS des autres drones visibles sur la Rade de Marseille
ℹ️
Les ports NMEA TCP sont : 5001 → Drone 1, 5002 → Drone 2, 5003 → Drone 3, 5004 → Drone 4.
